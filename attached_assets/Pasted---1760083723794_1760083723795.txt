Мы приняли стратегическое решение о "тотальной перезагрузке" продакшен-сервера. Мы отказываемся от попыток сохранить старые данные и хотим развернуть текущую, идеально работающую версию проекта из Replit на абсолютно чистый сервер.

**Наша Глобальная Цель:** Создать надежный, повторяемый процесс развертывания, который гарантирует, что сайт на сервере будет 1-в-1 копией проекта в Replit, и чтобы будущие обновления проходили гладко.

**Контекст, который ты должен знать:**
1.  **Сервер:** У нас есть VDS на TimeWeb. Мы управляем им по SSH.
2.  **Развертывание:** Мы используем PM2 для запуска приложения.
3.  **База данных:** PostgreSQL. Перед новым развертыванием мы будем **полностью удалять и пересоздавать базу данных с нуля**.
4.  **Процесс обновления:** В будущем мы будем обновлять сайт по схеме: Replit -> GitHub -> Сервер (через `git pull` и `pm2 restart`).

**Твоя задача: Подготовить проект к этому процессу и предоставить финальную инструкцию.**

**АЛГОРИТМ ДЕЙСТВИЙ:**

**1. Проанализируй и финализируй "сидинг"-скрипт (`seeding script`).**
*   **Проблема:** При чистом развертывании база данных будет пустой. Нам нужно, чтобы она автоматически наполнялась **минимально необходимыми данными** для старта.
*   **Задача:** Проверь и доработай `prisma/seed.ts` (или создай его, если его нет). Этот скрипт должен автоматически создавать:
    *   **Базовые курсы валют** (TJS, USD, EUR, RUB).
    *   **Базовые категории туров** ("Экскурсии", "Горные туры" и т.д.).
    *   **Базовые страны и города** (Таджикистан, Душанбе, и т.д.).
    *   **Одного администратора** по умолчанию (например, `admin` / `12345`), чтобы можно было сразу войти в админку.
*   Убедись, что команда `npx prisma db seed` корректно запускает этот скрипт.

**2. Проанализируй и финализируй процесс сборки (`build process`).**
*   **Проблема:** Чтобы избежать ошибок "на сервере не работает", мы должны перейти от запуска "сырого" TypeScript-кода через `ts-node` к запуску **скомпилированного** JavaScript-кода.
*   **Задача:**
    *   Проверь файл `tsconfig.json` и убедись, что он правильно настроен для компиляции всего из `src/` в `dist/`.
    *   Проверь, что в `package.json` есть скрипт `"build": "tsc"` (или аналогичный).
    *   Проверь, что `ecosystem.config.js` настроен на запуск **скомпилированного** файла: `script: "./dist/index.js"`.

**3. Создай финальную, пошаговую инструкцию по "Тотальной Перезагрузке".**
*   Напиши для меня инструкцию, которую я должен буду выполнить на сервере. Она должна быть максимально простой и состоять из команд, которые нужно вводить по одной.
*   **Инструкция должна включать в себя следующие этапы:**
    1.  **Подготовка:** Остановка старого сайта, удаление старой папки проекта, удаление и пересоздание базы данных.
    2.  **Установка:** Клонирование чистого репозитория с GitHub, установка зависимостей (`npm install`).
    3.  **Настройка:** Создание файла `.env` с необходимыми переменными.
    4.  **Сборка:** Компиляция TypeScript в JavaScript (`npm run build`).
    5.  **Первый запуск и "посев" данных:** Создание таблиц в базе (`npx prisma db push`) и наполнение их начальными данными (`npx prisma db seed`).
    6.  **Финальный запуск:** Запуск сайта через PM2 и `ecosystem.config.js`.

**Цель:** После твоей работы у меня на руках должен быть полностью готовый к чистому деплою проект и foolproof-инструкция, которая гарантирует 100% результат.
