Мне нужно доработать сайт, добавив функционал для B2B-партнерства с турагентами. Система должна состоять из трех частей: публичной страницы, раздела в админ-панели и личного кабинета для турагента.
Часть 1: Публичная страница "Партнерство для Турагентов".
Создай новую страницу на сайте для привлечения турагентов.
На этой странице размести форму заявки на партнерство со следующими полями: ФИО (полное), Гражданство, Адрес, Телефон, Email, поле для загрузки файла "Документ, удостоверяющий личность", поле для загрузки файла "Другие документы".
Добавь два обязательных чекбокса: "Согласен с Договором о партнерстве" и "Согласен с Политикой конфиденциальности".
После отправки формы, все данные и файлы должны сохраняться и отображаться в админ-панели в новом разделе.
Часть 2: Админ-панель.
Создай в админ-панели новый главный раздел "Турагенты".
Внутри этого раздела создай два подраздела: "Заявки" и "Турагенты".
В "Заявках" должен отображаться список всех новых заявок на партнерство. У администратора должна быть возможность просмотреть полную информацию по каждой заявке и прикрепленные файлы.
Для каждой заявки должны быть две кнопки: "Одобрить" и "Отклонить".
При нажатии на "Одобрить":
Создается учетная запись для нового турагента с ролью "Турагент".
Генерируется уникальный ID (формат: BT000001).
Генерируется надежный пароль.
На email турагента отправляется письмо с его логином (email), паролем и ссылкой для входа в кабинет.
Заявка перемещается в раздел "Турагенты".
При нажатии на "Отклонить", заявка удаляется.
В разделе "Турагенты" должен быть список всех одобренных партнеров с их основной информацией.
Часть 3: Личный кабинет Турагента.
Создай страницу личного кабинета, доступную только для авторизованных турагентов.
В кабинете должны отображаться: ID турагента, меню навигации.
Основная функция кабинета: "Форма заявки на тур" для регистрации приведенных клиентов. Поля: Название тура, Дата заявки, Рег. № заявки, Даты реализации тура, Кол-во туристов, Список туристов (ФИО).
После отправки этой формы, данные должны попадать в админ-панель для учета.
Также в кабинете должна быть таблица "Список реализованных туров", где агент видит историю своих заявок.
